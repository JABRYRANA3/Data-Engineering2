Data Engineering 2 


Piste Thématique Choisie
Pour l'ensemble de ce cours, on a choisi de travailler sur la Track A — Esport (OpenDota).
Toutes les données explorées (matchs, héros, archétypes, impact des patches) tournent autour de cette thématique. Cette même base de données a servi de fil rouge pour la totalité des Labs (1 à 3), des Assignments et du Projet final.

Structure du Répertoire
Ce dépôt est organisé par dossiers pour chaque étape du cours.

Dans chaque dossier de Lab Practice et d'Assignment, vous trouverez systématiquement :
Le Notebook complet contenant le code exécuté.
Le fichier CSV correspondant aux données générées.
Les métriques Spark UI justifiant les performances de l'exécution.

Les Rapports
Rapport des Labs Practice : Vous y trouverez les captures d'écran des Spark UI, ainsi qu'une section détaillant l'utilisation de l'IA (prompts et aide à la compréhension).
Rapport des Assignments : Construit sur le même modèle que le précédent (captures d'écran Spark UI et les explications et l'usage de l'IA)
Rapport du Projet Final : La synthèse complète et détaillée du projet de fin de module, basé sur les données Esport (OpenDota).
